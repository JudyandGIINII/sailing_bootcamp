# Training Ground completion matrix — synthetic prototype evidence

## Scope and status vocabulary

This is a repository-evidence matrix for the personal, browser-local Training Ground. It records what the current prototype exposes or records; it is not an acceptance of a lesson’s semantic content.

- `IMPLEMENTED_PROTOTYPE` — present in the local prototype and covered by the cited source/test evidence.
- `DECLARED_SYNTHETIC_ONLY` — a manifest/UI declaration is intentionally synthetic and unvalidated; it is not a calculated real-world value.
- `NOT_IMPLEMENTED` — no implementation supplies the described physical, navigational, safety, or correctness behavior.
- `NOT_APPLICABLE` — the facet does not apply to the lesson. No facet below is marked this way because all six are part of each lesson surface.

All lessons use `training-sloop-v1`, registered lesson action policy, and the shared synthetic scenario/replay shell. The visible boundary is **“Simulation-only prototype • Unvalidated content • Not navigation, safety, or certification guidance.”** (`tests/smoke/app.spec.ts`, `runs the keyboard-only L01 prototype with visible non-navigation status`).

## Limits

This matrix does **not** establish real sailing physics, safety, navigation authority, certification, supported-browser coverage, domain validation, or release eligibility. All lesson validation dispositions remain `assumption`; the matrix does not promote the registry or change release gating. It also does not convert L05 record labels into recommendations, correctness outcomes, or safety results.
