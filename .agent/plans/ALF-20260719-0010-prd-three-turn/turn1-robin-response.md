## 1) 한 문장 총평
전반적으로 제약 합의(데스크톱 우선/입문 대상/Training Sloop 단일/MVP 5레슨/비인증·비실시간 API·약한 게임성)는 유지되고 있으나, 결정론적 검증·좌표/단위·L01–L05 채점 루브릭·접근성·출시 게이트의 정량성이 불완전하여 MVP 완료성 평가가 어렵다.

## 2) 개선 발견사항 (심각도 순)

1. [Critical] `섹션 8.2, FR-01, 수용 기준 1, 섹션 10`
   - **문제/누락:** 동일 입력 재현 조건은 제시되어 있으나(시나리오·seed·input log·모델/프로필 버전), 물리 계산의 **고정 틱(Δt), RNG 시드 처리, 부동소수점 정규화, 업데이트 순서, 이벤트 정렬 규칙**이 정의되지 않았다.
   - **영향:** 브라우저/성능 환경에 따라 같은 조건에서도 상태/채점 차이가 발생해 golden replay로 회귀 검증이 성립하지 않는다.
   - **바로 반영 문구:** `시뮬레이션은 고정 시간 스텝(예: 0.1초) 기반으로 진행하며, 모든 물리·AI·난수 연산은 정수 정밀도 고정(또는 결정론용 정밀도 정책) 후 처리한다. replay 입력 로그는 (tick, 이벤트타임스탬프, 입력 순서) 해시와 함께 저장되며, 허용 오차(예: 위치 0.2m, 속도 0.05kn, 각도 0.5°) 내에서 동일 판정/점수를 만족해야 한다.`

2. [Critical] `섹션 8.1, 8.2, FR-04`
   - **문제/누락:** 바람·조류·조석·파도·heading/COG의 **좌표계/기준(진북/자북/자기북), 단위(kn, m/s, deg), 방향 회전 기준, 시간대/시간 기준(시각 vs dt)**가 미정의다.
   - **영향:** 사용자 인지형 피드백(`apparent wind`, `drift trail`, `STW/SOG`)이 실제 의사결정 인과와 어긋나고, 환경 변화가 좌우 반대로 해석될 위험이 있다.
   - **바로 반영 문구:** `모든 벡터는 북미터-동미터 기준(ENU 또는 NED 중1종) 좌표계와 0°=북/시계방향 규약을 명시하고, 각도는 deg, 속도는 kn, 거리 m(또는 NM)으로 통일한다. heading·COG·wind/course는 모두 같은 좌표/기준으로 정규화 후 계산한다.`

3. [High] `섹션 7.2(L01–L05), FR-06, FR-07, 섹션 7.3`
   - **문제/누락:** L01–L05에 대해 **관측/판단/행동/안전회복/결과**를 묶는 정량적 채점 루브릭이 없다. “안전회복”은 서술되나 실행 조건/최소점수/패널티 구조가 부재.
   - **영향:** 레슨별 합격/미달을 객관적으로 판단할 수 없어 C1 완료 판정과 파일럿 비교가 불명확하다.
   - **바로 반영 문구:** `각 레슨에 대해 O(관측), J(판단), A(행동), R(안전회복), P(진행성공)의 하위 점수와 가중치(예: O30/J30/A20/R20), 실패-회복 규칙(예: 위험 이벤트 후 2개 행동 이내 회복 시 회복점수 부여, 미회복 시 F)을 명시하고, FR-06에서 레슨별 pass 조건(최소 총점, 치명적 오차 조건)을 정의한다.`

4. [High] `섹션 10 수용 기준 9, 10`
   - **문제/누락:** 입문 수강생 파일럿이 언급되나 **사전/사후 측정 지표와 게이트 실패 기준**이 없음.
   - **영향:** “파일럿 통과”의 의미가 추상적이어서 출시 게이트 자동화가 불가하고, 교육 효과를 입증 못한 상태로 릴리즈 리스크가 남는다.
   - **바로 반영 문구:** `런치 게이트를 `사전 이해도(개념 퀴즈 정확도), 조작 적합도(안전 위반율), 복기 품질(인과 설명 정답율), 회복 성공률` 4개 지표로 수치화(예: 각 항목 최소 75%)하고, 1개 미만이면 게이트 실패로 규정한다.`

5. [High] `섹션 6.3, 120, FR-09, 섹션 10 8`
   - **문제/누락:** 영종도·덕적도 관련 정적 콘텐츠에서 오인 방지 조치가 “고지” 수준에 머물고, **UI/UX에서 지속적 라벨링 및 탐색 제한**이 빠져 있다.
   - **영향:** 학습자가 “실제 항법 자료”로 오해해 실제 조치로 전이할 가능성이 있어 안전·윤리 리스크가 높다.
   - **바로 반영 문구:** `모든 한국 연안 맥락 시나리오 화면·맵·결과 리포트에 상단 고정 배너(“학습 시뮬레이션 모형, 실제 항법·해도·예보 아님”)와 장소 오인 방지 문구를 의무 노출하고, 시나리오 저장/공유 카드에 동일 문구를 자동 포함한다.`

6. [Medium] `섹션 10-1~2, 섹션 196`
   - **문제/누락:** 브라우저 성능 요건이 “승인 브라우저·성능 예산은 기술 옵션에서 확정”으로 미정이고, **60fps/틱 처리량/메인스레드 한계** 같은 최소 기준이 없다.
   - **영향:** 데스크톱 우선 브라우저에서도 저성능 장치에서 시뮬레이션 지터가 발생해 판단 루프가 깨지고 재현성이 악화된다.
   - **바로 반영 문구:** `출시 게이트에 `최소 60fps 유효 구간(예: 50% 구간 이상), 시뮬레이션 틱 지연 ≤ 16.7ms(상대 허용치), HUD 갱신 레이턴시 상한, 메모리 상한`을 명시하고 성능 계측 로그를 저장한다.`

7. [Medium] `섹션 190-196`
   - **문제/누락:** 접근성 요건이 키보드/색상 구분/일시정지·속도조절로 제한되어 있어, **초점 순환, ARIA 라벨, 스크린리더 상태 안내, 동작 단축키 충돌/옵션, 고대비·감속 모드** 등 필수 항목 부재.
   - **영향:** 데스크톱 사용자 접근성 실패와 입력 실수(조작 착각)로 교육효과가 감소.
   - **바로 반영 문구:** `키보드 포커스 순서, ARIA 롤/라벨, 상태 라이브 영역, 폴리시(키 입력 조합) 노출, reduced-motion 모드, 입력 오류 회복(UNDO/재매핑) 정책을 비기능 요구사항에 추가한다.`

8. [Medium] `섹션 2, 25-35, 228-230`
   - **문제/누락:** 조사 근거 링크는 존재하나, **설계 변수별 데이터 출처·신뢰수준·버전/수집일 메타데이터가 요구사항에 직접 연결되어 있지 않다**(예: 계수값·패널티 계수·임계값 유래).
   - **영향:** 모델/콘텐츠 변경 시 provenance 추적이 어렵고, 검수 단계에서 “왜 그 값인지”를 재현성 있게 증명하기 어렵다.
   - **바로 반영 문구:** `모든 보트/환경 파라미터(폴라, 안전 임계값, 가중치, 난이도 보정)의 `source_id + version + 근거 링크/담당자`를 요구사항 항목에 필수 필드로 두고, FR 문항에 참조 누락 시 구현 금지 조건으로 둔다.`

## 3) UNVERIFIED — 도메인 검증 필요

- `Training Sloop`의 선체·리그 단순 합성 모델이 **입문 교육 목적에서 충분한 인과 신뢰성**을 갖는지(과대/과소 단순화 여부) — 현재는 강사/도메인 전문가 검증으로만 처리 필요.
- `reef/감속/회항 선택이 항상 “안전 행동으로 정답”이라는 가중치 설계`가 실제 교육적 적절성 측면에서 정합한지.
- `파도·시정·조류·조위`의 페널티/한계 임계값(예: 언제 위험/회복 점수 부여) 적정성은 해양 실습 경험자가 검토해야 함.
- 한국 연안 명칭 기반 정적 콘텐츠가 사용자에게 **오해 없는 형식/범위로 전달되는지**에 대한 최종 판단은 해양 안전 교육 전문가 검토 필요.
